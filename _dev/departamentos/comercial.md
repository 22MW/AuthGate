## Resumen humano

AuthGate ya tiene release production `1.1.1` publicada y puede empezar fase comercial. El posicionamiento recomendado no es “plugin de login”, sino “experiencia de acceso frontend para WordPress, WooCommerce, membresías, academias y zonas privadas”.

## Descubierto

- Valor principal: sustituir la experiencia fría de `wp-login.php` por formularios frontend personalizables.
- Público natural: webs con zona privada, tiendas WooCommerce, academias, membresías, intranets ligeras, portales de clientes y sitios con contenido restringido.
- Diferenciadores reales tras `1.1.1`:
  - Login, registro, recuperación y protección de páginas en frontend.
  - Integración WooCommerce.
  - Registro configurable desde AuthGate.
  - Textos editables desde pestaña propia.
  - WYSIWYG bajo logo/cabecera.
  - CSS propio con presets blanco/oscuro y editor CodeMirror.
  - Popup shortcode con `label` personalizable.
  - Mail Mint condicional.
  - Release GitHub con ZIP y updater corregido para futuras versiones.

### Posicionamiento base

AuthGate convierte el acceso de WordPress en una experiencia frontend cuidada, personalizable y alineada con la marca, sin mandar al usuario a pantallas técnicas de WordPress.

### Público objetivo

- Dueños de tiendas WooCommerce con área de cliente.
- Creadores de academias o membresías.
- Agencias WordPress que quieren entregar login/registro más profesional.
- Negocios con zona privada para clientes.
- Proyectos que necesitan restringir páginas sin montar un sistema complejo de membership.

### Problemas que resuelve

- El login nativo de WordPress se ve técnico y poco profesional.
- `wp-login.php` rompe la experiencia visual de la web.
- Personalizar textos, registro y estilos suele requerir código o varios plugins.
- WooCommerce y WordPress tienen ajustes de registro repartidos.
- Muchas webs necesitan proteger páginas simples sin instalar un sistema pesado.

### Propuesta de valor

“Crea una experiencia de acceso profesional en WordPress: login, registro, recuperación, páginas protegidas y estilo visual desde el frontend, sin depender de `wp-login.php`.”

### Beneficios principales

- Mejor primera impresión para usuarios registrados.
- Acceso integrado visualmente con la marca.
- Menos fricción en registro y login.
- Mejor control sobre textos, estilos y comportamiento del registro.
- Útil tanto para WordPress puro como WooCommerce.
- Menos dependencia de snippets o maquetación manual.

### Ángulos de campaña

1. **Dolor visual**
   - Problema: el login nativo de WordPress parece técnico.
   - Promesa: acceso frontend bonito y personalizable.

2. **WooCommerce**
   - Problema: registro y acceso de cliente poco controlados.
   - Promesa: registro/login frontend más profesional y coherente.

3. **Zonas privadas**
   - Problema: proteger contenido suele requerir plugins grandes.
   - Promesa: restricción simple con login frontend integrado.

4. **Agencias**
   - Problema: cada cliente pide “que el login no parezca WordPress”.
   - Promesa: solución reutilizable, configurable y entregable.

5. **Marca y confianza**
   - Problema: mandar a una pantalla técnica reduce confianza.
   - Promesa: mantener al usuario dentro de la experiencia visual de la web.

### Objeciones y respuesta

- “Ya tengo WooCommerce.”
  - AuthGate no sustituye WooCommerce; mejora la experiencia frontend y respeta su flujo.

- “Puedo hacerlo con código.”
  - Sí, pero AuthGate lo centraliza en ajustes y evita repetir snippets por proyecto.

- “Uso un constructor visual.”
  - AuthGate se integra mediante shortcodes y permite estilo propio.

- “No necesito membership completo.”
  - AuthGate cubre acceso y protección ligera sin añadir complejidad innecesaria.
