Resumen humano

AuthGate tiene release pública 1.2.0 publicada. En rama de trabajo ya están aplicados MS0, MS1, MS2 y MS3 con fixes del primer QA multisite. Queda repetir QA conjunto antes de avanzar. Últimos fixes: Estilo global visible, `/wp-admin` redirige al login AuthGate y sites enlazan a config global.

Problema confirmado

  • MS1: Registro se gestiona desde la red y usa la política nativa multisite.
  • MS2: Textos ya tienen fallback sitio → red → default.
  • MS3: CSS ya permite heredar, sobrescribir o desactivar por sitio.
  • Exclusiones y Mail Mint sí están por sitio.
  • General mezcla scopes globales y por sitio.
  • La red debe mandar si bloquea registro de usuarios.

Decisión base

Modelo recomendado: sitio → red → default. La red define política y defaults; cada web puede sobrescribir Textos/Estilo/Registro cuando la red lo permite.

Validación necesaria

  • Red activa/desactiva registro desde AuthGate.
  • Sites no muestran control local de registro.
  • Sitio A/B muestran textos distintos.
  • Sitio A/B aplican CSS distinto o heredado.
  • Exclusiones no se cruzan entre sitios.
  • Mail Mint no se cruza entre sitios.
  • Network admin no muestra pestañas Textos/Estilo.
  • Site admin respeta diseño 22MW.

Riesgos

  • Slugs por sitio pueden requerir flush de rewrites por subsite.
  • Cambiar scopes sin fallback puede ocultar valores existentes.
  • WooCommerce puede variar entre subsites.
